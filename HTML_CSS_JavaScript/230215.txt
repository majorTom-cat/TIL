브라우저 엔진 : 웹페이지를 해석해서 화면에 뿌려준다

웹 : client의 요청(request)과 server의 응답(response)과정

도메인 : 주소를 보기 편하게 하기위한 서비스

인터넷 주소는 크게 3가지
MAC Address
IP Address : 목적지 컴퓨터 식별(사설 아이피 : C클래스) 192.169.0.2~6 [IPv4 : 32 bit]
Port Address

서브넷마스크 : 255.255.255.0

라우터 테이블에 컴퓨터의 실제 IP가 등록
IP는 같은 네트워크에 있다는것만 알려줌

동일한 네트워크 안에서는 MAC Address 사용. 유무선 Lan카드사용. 고유한 주소를 MAC Address라고 함

IP에 서브넷마스크 & 연산. 결과를 통해 같은 네트워크에 있는지 알수있다.

데이터를 허브가 받아서 모든 컴퓨터에 뿌린다. 목적지 주소가 같으면 받고 다르면 버림

DNS 도메인 네임 시스템 혹은 도메인 네임 서버
도메인 네임 -> IP Address로 변환
보통 우리가 쓰는 컴퓨터는 DNS Server가 없음. 다른 DNS를 쓸 수 있게 셋팅은 되어있음

네트워크가 다르면 라우터를 통해 경로를 찾음(라우팅)

내가 요청한 서비스가 어디쪽인지 port로 찾음. 보통 웹은 80번 씀
